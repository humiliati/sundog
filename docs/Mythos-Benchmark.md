## Maintainer Note: Source Conflation, Frozen Legacy, and Future Sundog Delineation

This report is being preserved substantially intact as an artifact of external model interpretation. It is useful precisely because it shows how a powerful outside system attempted to read the public Sundog material, connect it to historical computing, and synthesize a larger story. Some of that synthesis is productive. Some of it is wrong.

The central correction is this:

**The `humiliati/sundog` project is not currently a port, fork, preservation layer, or p-system continuation of *SunDog: Frozen Legacy*.**

The Atari ST / P-System material in this report appears to come from a different public project: `laanwj/sundog`, an open-source port of the Atari ST game *SunDog: Frozen Legacy*. That project really does involve a p-system interpreter, SDL2, OpenGL ES 2+, Atari ST gameplay preservation, disk-image handling, save/load shortcuts, and related emulator-port details. The mistake is not that *SunDog: Frozen Legacy* is fictional. The mistake is that this report merged that separate historical/game-preservation project into the architecture of this repository.

This repository’s current Sundog work is organized around a different claim: **Alignment Without Sight**. The active pattern is not “retro software preservation as alignment substrate,” but hidden-state control through indirect signatures. The existing and proposed workbenches follow a recurring structure:

1. the decisive state is hidden;
2. the world leaks usable structure through shadows, wakes, traces, gradients, pressures, delays, distortions, or partial reactions;
3. an agent acts from those traces rather than from privileged world-state access;
4. the operating envelope and failure boundary are made explicit.

That distinction matters. The Gimmicks ledger states the public posture cleanly: Sundog should not be framed as “better game AI,” “beats AlphaGo,” or “searches harder.” The intended contrast is narrower and cleaner: perfect-information systems master visible boards; Sundog asks what remains possible when the board is denied.

### What this report got right

The report recognized several real themes in the project:

- Sundog is concerned with control under occlusion.
- The useful object is often an indirect or displaced signature rather than a directly observed target.
- The work should be judged by bounded demonstrations, operating envelopes, and failure surfaces rather than broad universal claims.
- Game-like workbenches can help make the idea publicly legible, provided they are not sold as ordinary game-winning AI.

Those readings are close enough to be worth preserving.

### What this report got wrong

The report incorrectly treats Atari *SunDog: Frozen Legacy* preservation as one half of this repository’s technical architecture. That produces several downstream errors:

- It describes a p-machine emulator as if it lives in this repo.
- It treats 64KiB / 128KiB p-system constraints as if they were part of the Sundog Alignment implementation.
- It interprets Atari-era “frozen systems” as a deliberate research substrate for this project.
- It implies that modern Sundog work and the historical FTL game are technically continuous.
- It lets the phrase “frozen legacy” overtake the actual claim boundary of the project.

This is source conflation, not a discovered dependency.

1. **Sundog Alignment**  
   The active research/programming work in this repository: indirect signatures, hidden targets, browser/headless workbenches, bounded claims, baselines, and failure sweeps.

2. **Historical SunDog: Frozen Legacy**  
   The 1980s FTL game and its independent modern preservation/port ecosystem. This is relevant as name resonance, retro-computing context, and possible public metaphor, but not as current project architecture.

3. **Sundog Gimmicks**  
   Proposed game-native demonstrations such as Pressure Mines, Shadow Fleet, Occluded Code, Shadow Shoe, Scent Snake, Blind Labyrinth, and later spectacle workbenches. These are not retro ports. They are controlled demonstrations of action from denied state.

4. **Possible Atari Sundog Gimmick**

Such a workbench would need the same discipline as the others: hidden target, indirect signal, transformation, actionable output, baseline, and failure boundary.
