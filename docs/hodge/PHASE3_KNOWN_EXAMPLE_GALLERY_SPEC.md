## 1. What this spec is, and what it is not

This freezes **which** known examples a future gallery may use, **how** each row
must be labeled, and the **falsifiers** that retire the gallery. It deliberately
stops short of building anything. Phase 2 (`HODGE_PHASE2_BRIDGE_NOTE.md`)
licenses *only* this spec, and *only* if every example carries a "known
because…" field and a "does not imply…" field.

It is **not**: a detector, a claim about any open case, evidence for the
conjecture, or a rendered/visual artifact. The first three are forbidden by the
lane; the fourth is a Phase 4 decision, not this spec's to make.

## 2. Design rules (binding on every row)

1. **Register-ladder backbone.** Each row is read through the reader's R1–R4
   ladder ([`../HODGE_READER_NOTE.md`](../HODGE_READER_NOTE.md) §3): the **body**
   is an algebraic cycle (R4); the **shadow** is a rational `(p,p)` class (R3);
   the row never conflates either with a harmonic representative (R2) or a
   differential form (R1).
2. **Two mandatory prose fields.** Every row states **"known because…"** (the
   theorem that makes the body algebraic) and **"does not imply…"** (the fence:
   what a non-specialist must *not* infer).
3. **Rational only.** Every class is a *rational* Hodge class unless a row is
   explicitly tagged "integral variant" (and integral rows are excluded from the
   known roster — see §5).
4. **Known-only roster.** A row may appear in the roster *only* if its
   algebraicity is a theorem. Open or unknown cases never appear in the roster;
   if shown at all (Phase 4), they live in a separately labeled "open" panel,
   never beside the known rows.
5. **Inherit the reader's fences.** §6 of the reader (rational≠integral,
   projective≠Kähler, type≠algebraic, Hodge-loci≠construction, reads-not-proves)
   applies verbatim.

## 3. Row schema

Each roster entry must fill:

| Field | Meaning |
| --- | --- |
| Example | the variety and the cohomology class in question |
| Body | the algebraic cycle (R4) |
| Shadow | the rational `(p,p)` class (R3) |
| codim `p` | the codimension |
| Known because | the theorem certifying algebraicity |
| Does not imply | the fence against over-reading |
| Label | one of: `tautological` · `flagship` · `boundary-demonstrator` · `deferred` · `hard-excluded` |

## 4. Gallery roster (known, pre-registered)

Roster at a glance, ordered by increasing richness:

- **G1** — projective space `Pⁿ` — `tautological warm-up`
- **G2** — points on a curve — `trivial entry`
- **G3** — divisors via Lefschetz `(1,1)` — `flagship`
- **G4** — curves on a surface — `concrete instance of G3`
- **G5** — codim-two classes on a threefold — `rational/integral boundary-demonstrator`

### G1 — Projective space `Pⁿ` (hyperplane powers)

- **Body:** a codimension-`k` linear subspace `P^{n−k}` (and `ℚ`-multiples).
- **Shadow:** the generator of `H^{2k}(Pⁿ,ℚ) = ℚ`, of type `(k,k)`.
- **codim `p`:** `k`, for `0 ≤ k ≤ n`.
- **Known because:** each `H^{2k}(Pⁿ)` is one-dimensional and spanned by the
  linear-subspace class; the whole cohomology ring is algebraic (generated by
  the hyperplane class).
- **Does not imply:** anything about varieties with richer cohomology. `Pⁿ` has
  *no* non-algebraic Hodge classes by triviality, not by a detector.
- **Label:** `tautological warm-up`.

### G2 — Points on a curve

- **Body:** a point (a `0`-cycle) on a smooth projective curve `C`.
- **Shadow:** the generator of `H²(C,ℚ) = ℚ`, of type `(1,1)`.
- **codim `p`:** `1` (`= dim C`).
- **Known because:** `H²` of a curve is spanned by the point class; codimension
  one on a curve is just points, manifestly algebraic.
- **Does not imply:** that curves test the conjecture. The interesting cohomology
  of a curve is `H¹`, of type `(1,0)⊕(0,1)` — *not* `(p,p)` — so there is no
  middle Hodge `(p,p)` class to ask about.
- **Label:** `trivial entry` (the `dim = 1` base case of G3).

### G3 — Divisors via the Lefschetz `(1,1)` theorem (FLAGSHIP)

- **Body:** a divisor `D` (a codimension-one subvariety, or `ℚ`-combination) on
  any smooth projective `X`.
- **Shadow:** a rational `(1,1)` class in `H²(X,ℚ) ∩ H^{1,1}(X)`.
- **codim `p`:** `1`, in every dimension.
- **Known because:** the **Lefschetz `(1,1)` theorem** — via the exponential
  sequence, `c₁` realizes every integral (hence rational) `(1,1)` class as the
  class of a divisor. This is the `p = 1` case, known in all dimensions.
- **Does not imply:** anything in codimension `≥ 2`. The mechanism (`c₁` of line
  bundles) has *no* higher-codimension analogue; this is exactly where the
  elementary part of the conjecture stops.
- **Label:** `flagship` — the cleanest honest known case.

### G4 — Curves on a surface (concrete instance of G3)

- **Body:** a curve `C ⊂ S` on a smooth projective surface `S` (dimension `1`,
  codimension `1`).
- **Shadow:** its `(1,1)` class; by Lefschetz `(1,1)`, the rational `(1,1)`
  classes are `NS(S)⊗ℚ`.
- **codim `p`:** `1`.
- **Known because:** the Lefschetz `(1,1)` theorem, specialized to a surface.
- **Does not imply:** anything in higher dimension. Keep Néron–Severi rank and
  the Hodge index theorem *out* of first copy — true, but a different subtlety
  that distracts from the body/shadow point.
- **Label:** `concrete instance of G3`.

### G5 — Codimension-two classes on a threefold (rational/integral boundary)

- **Body:** a curve (codim `2 = dim 1`) on a smooth projective threefold `X`.
- **Shadow:** a rational `(2,2)` class in `H⁴(X,ℚ) ∩ H^{2,2}(X)`.
- **codim `p`:** `2` (`= n−1` for `n = 3`).
- **Known because:** hard Lefschetz makes codimension `n−1` the dual of
  codimension one, so the *rational* Hodge conjecture holds in this range.
- **Does not imply:** the **integral** statement. Integral Hodge can *fail* here
  (Kollár-type examples); Voisin's positive integral results are special to
  uniruled or Calabi–Yau threefolds (`arXiv:math/0412279`). **Hard fence:
  rational only.**
- **Label:** `boundary-demonstrator` — the row that teaches the rational/integral
  line, and the most likely to be mis-stated, so it carries the heaviest fence.

## 5. Deferred and hard-excluded

### Deferred (admissible only after an expanded lit pass + expert review)

- **Abelian varieties.** Exotic Hodge classes (Weil classes; Mumford–Tate
  structure) live here — exactly where rational Hodge becomes subtle and
  open-adjacent. Defer.
- **K3 surfaces.** Divisor classes are fine by Lefschetz `(1,1)`, but the
  Hodge-theoretically interesting content is the transcendental lattice. Too
  subtle for a first gallery.
- **Products.** Nontrivial product examples can quickly introduce the
  abelian-variety / Weil-class story; defer with abelian varieties unless a later
  lit pass isolates a completely elementary product row.
- **Cubic fourfolds.** Dimension four, codimension two, and special enough to
  require its own literature pass (including known special results); hard defer
  so the first gallery does not blur "general fourfold open" with special
  fourfold families.

### Hard-excluded (never in the known roster)

- **General fourfold, codimension two** (`H⁴ ∩ H^{2,2}` on a `4`-fold). This is
  the first dimension where the general statement is not known (Clay's
  "dimension four is unknown" boundary). A known-example gallery must never
  display it as if settled — that is the inverse of `HODGE-TOY-LAUNDERING`. If
  shown at all (Phase 4), it is labeled **OPEN** and lives outside the known
  roster.
- **Any integral Hodge claim.** Integral is a separate, false-in-general variant;
  it never appears in the rational gallery without an explicit "integral variant —
  can fail" label.
- **Compact Kähler (non-projective).** Voisin's counterexample; excluded.

## 6. Pre-registered falsifiers

- `HODGE-TOY-LAUNDERING` (primary): a row, label, or later visual presents a
  known example as evidence for an open case — or an open case appears without an
  explicit OPEN/excluded label.
- `HODGE-CATEGORY-ERROR`: a row conflates the body (cycle), the shadow (rational
  `(p,p)` class), a harmonic representative, or a differential form — i.e. it
  breaks the R1–R4 ladder.
- `HODGE-LIT-MISMATCH`: a "known because" cites the wrong theorem or misstates a
  case (e.g. asserts integral where only rational holds, or lists an open case as
  known).
- `HODGE-VISUAL-MISCALIBRATED` (forward guard for Phase 4): if the gallery is
  ever rendered, no visual may make the cycle look directly readable off the form.

## 7. Exit criterion and what this licenses

Exit criterion (matches the charter Phase 3 exit): every roster row carries all
schema fields including "does not imply"; no open, integral, or Kähler case
appears in the known roster; the four falsifiers above are pre-registered; the
reader's §6 rational/projective fences are inherited.

This spec licenses **only** a Phase 4 *decision* — whether the gallery becomes a
public page, a section of a broader math rail, or stays an internal docs note —
and that decision may be taken only after external Hodge-theory review. It does
**not** by itself authorize building any visual or public artifact.
